Christopher D. Pang
Current checkpoint: July 2026

PURPOSE

This folder collects the main public specifications, companion branches, integration drafts,
measurement tools, and formal-methods experiments developed around Primitive Axiom Layers
(PAL), PECAN, PEA, and Seed, Not Feed.

The collection is intended to help a human reader or AI system understand the framework as a
versioned body of work rather than as a single flattened document.

CORE READING ORDER

1. PPP Kernel v0.3
   Start here for the architectural map and the end-to-end route.

2. PAL Spine v1.3
   Canonical structural specification.
   Controls questions about distinction, trace, persistence, boundaries, relations, cadence,
   constraint, and typed closure.

3. PECAN v1.0
   Canonical consequential-crossing and authorization protocol.
   Controls questions about when description is being used to route consequence, how authority
   is declared, and how wording drift, repetition, or compression must not manufacture permission.

4. PEA Core v1.0
   Canonical public ethical-review specification.
   Controls questions about burden, standing, proportionality, contest, remedy, refusal,
   reopening, and accountable human judgment.

5. Seed, Not Feed / SEED
   Human-facing release discipline.
   SEED = Sustainable, Empathic Expression and Disclosure.
   The goal is to communicate useful results without erasing uncertainty, consuming agency,
   or filling all remaining thinking space.

PRIMARY ROUTE

PAL description
    -> PECAN consequential crossing and lineage
    -> optional PEA ethical evaluation
    -> accountable human or institutional decision
    -> SEED-facing explanation, correction, or suggested route

Information may travel backward for correction, audit, or reopening.
Authority may not silently travel backward and rewrite the evidence that supposedly justified it.

CANONICAL AUTHORITY

When documents disagree inside a specific layer:

- PAL Spine v1.3 controls PAL structure.
- PECAN v1.0 controls consequential-crossing protocol.
- PEA Core v1.0 controls the declared PEA ethical review.
- PPP v0.3 explains integration but does not silently amend the canonical component documents.
- Newer canonical documents override stale terminology in older bundles or working drafts.

COMPANION DOCUMENTS

A0 Boundary Kernel v0.8.2
    Software-facing companion branch for unresolved-to-readable boundary handling.
    It is not software operating at A0 and does not replace PAL or Zero-State Axioms.

BLA in PAL v0.9.1
    Boundary-ledger accounting for abstraction, residuals, layer selection, and model-boundary repair.
    A residual may justify opening a provisional ledger; it does not automatically create a new law.

CLEF v1.0
    Cluster-Layer Entropy Focus.
    A declared measurement language for readable boundaries, coherence, cost, excess, recurrence,

Riemann Ledger v0.56
    Private formal-methods laboratory and resumable research scaffold.
    It demonstrates versioning, proof packets, typed burdens, tests, certificates, reopening,
    blocked promotion, and unresolved frontiers.
    It does NOT prove the Riemann Hypothesis and is not evidence that PAL is physically true.

PAL Public Bundle v0.1
    Earlier public synthesis and provenance record.
    Useful historically, but newer canonical documents control current terminology and architecture.

HOW TO USE THIS FOLDER WITH AN AI MODEL

Recommended instruction:

"Treat these files as a versioned corpus. Read PPP for the map, then defer to PAL v1.3,
PECAN v1.0, and PEA Core v1.0 inside their own layers. Treat A0 Boundary Kernel, BLA,
and CLEF as companion branches. Treat the Riemann Ledger as a laboratory and case study,
not as proof. Preserve unresolved claims, cite document and version, and do not let later
authority rewrite earlier evidence."

Do not simply flatten every document into one undifferentiated prompt. A capable system should:

- preserve version and document identity;
- distinguish canonical specifications from supplements and experiments;
- keep descriptive, protocol, ethical, and human-release authority separate;
- mark unresolved burdens instead of forcing closure;
- trace claims back to their source document;
- prefer newer canonical language over stale working terminology;
- avoid treating repeated wording as independent corroboration.

CORE DISCIPLINE

Description is not recommendation.
Recommendation is not permission.
Permission is not authorization.
Protocol validity is not ethical approval.
Ethical approval is not execution authority.
A model output is not an accountable human decision.
